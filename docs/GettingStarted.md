# Getting Started with PrepVerse

Welcome to the PrepVerse contributor documentation! This guide is designed to help you quickly understand the project, configure your local workspace, and make your first contribution.

---

## 🚀 Welcome to PrepVerse

PrepVerse is a modern, community-driven placement preparation ecosystem designed for students and career changers. We gamify and streamline the journey to securing roles in tech, finance, and engineering through structured curricula, AI-powered evaluation, and community challenges.

---

## 🗺️ Contribution Path

Whether you're looking to contribute as part of **Google Summer of Code**, **GSSoC**, **ECSoC**, or simply as an independent developer, here is our recommended roadmap:

### 1. Familiarize Yourself with the App
Run the application locally and explore each section:
- **Dashboard**: Track daily preparation progress.
- **DSA Module**: View, code, and evaluate standard data structure challenges.
- **Resume Studio**: Review resume tips and build structures.
- **Interview Hub**: Engage in conversational mock interviews evaluated by Gemini.
- **Aptitude Trainer**: Take time-boxed numerical and logical exams.

### 2. Read the Guidelines
Before writing code, make sure you've read and understood:
- [CONTRIBUTING.md](../CONTRIBUTING.md) — Coding styles, branch naming, and pull request flows.
- [CODE_OF_CONDUCT.md](../CODE_OF_CONDUCT.md) — Our community standards.
- [SECURITY.md](../SECURITY.md) — How to report security concerns.

### 3. Find an Issue
Head to the **PrepVerse Issue Tracker** and filter by:
- `good first issue` or `beginner` labels for simple scope improvements.
- Check our roadmap templates inside `roadmap/` for curated lists of 100+ issue ideas!

### 4. Claim the Task
Comment on the issue stating your interest. Once a maintainer assigns it to you, create a branch from the `develop` branch:
```bash
git checkout -b feature/issue-title
```

### 5. Submit your PR
Write clean, modular code, run the linter and compiler, and open a Pull Request against the `develop` branch following our [Pull Request Template](../.github/PULL_REQUEST_TEMPLATE.md).
